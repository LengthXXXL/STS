STS V1 技术架构建议

技术栈保持你的要求：Python 3.10+、MySQL 8.0、Vue 3。后端建议用 FastAPI，适合前后端分离、API 文档、异步任务和类型校验。

模块分层
Vue 3 前端

页面路由、积木画布、右侧积木库、回测图表、论坛、积木分享、管理后台
Python API 服务

JWT 鉴权、CRUD、权限校验、文件接口、审核接口、推荐接口
MySQL 8.0

用户、策略、积木、模拟账户、行情缓存、回测结果、论坛、推荐事件
行情适配器

AKShare / yfinance 按需获取 A股、美股 1分钟/5分钟 K线，并写入缓存
回测引擎

解析策略 JSON，注入行情/账户/市场规则，输出收益、回撤、成交明细
推荐服务

只服务积木分享页：热门、相似、个性化、新手精选、最新发布
文件存储

本地文件目录保存图片、附件、回测报告；数据库记录文件元数据和权限
后台任务

V1 可先用进程内任务或轻量任务队列；后续再升级 Redis + Celery

推荐架构取舍

1

FastAPI 后端

适合 REST API、JWT、自动接口文档、Pydantic 参数校验。比传统模板式后端更适合前后端分离。

2

回测引擎独立成服务层

不要写在控制器里。让 API 只负责接收任务，回测引擎负责策略解释、市场规则和收益计算。

3

行情按需获取 + MySQL 缓存

符合内存有限的约束。用户选股票和时间段后再获取数据，缺多少补多少。
