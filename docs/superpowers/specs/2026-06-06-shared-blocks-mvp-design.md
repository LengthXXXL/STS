# 积木发布与分享页 MVP 设计

## 目标

把已经完成的私有自定义积木闭环扩展成公开资产闭环：

用户在个人空间发布自己的积木，积木进入审核；管理员审核通过后，积木出现在“积木分享”页；其他用户可以搜索、查看、收藏并导入到自己的积木库。

本阶段只做积木分享，不做论坛发帖。论坛后续会复用这里产生的公开积木详情链接。

## 范围

本阶段包含：

- 个人空间“我的积木”增加发布入口。
- 发布后状态从 `private` 变为 `pending_review`。
- 管理员可以查看待审核积木并通过或拒绝。
- 审核通过的积木在积木分享页公开展示。
- 积木分享页支持关键词搜索、分类筛选、标签筛选、排序、分页。
- 公开积木详情展示作者、分类、标签、节点摘要、连接数量、收藏数、导入数。
- 登录用户可以收藏公开积木。
- 登录用户可以导入公开积木到自己的“我的积木”。
- 导入时复制模板内容到当前用户的私有自定义积木库，并避免同名冲突。
- 记录浏览、收藏、导入行为，作为推荐算法后续数据地基。

本阶段不包含：

- 论坛发帖、评论、帖子审核。
- 复杂个性化推荐模型。
- 公开积木版本历史。
- 批量审核和管理后台完整信息架构。
- 管理员精选配置页。

## 用户流程

### 发布积木

1. 用户进入个人空间的“我的积木”。
2. 用户点击某个私有积木的“发布”按钮。
3. 系统把该积木状态改为 `pending_review`。
4. 用户看到状态文案“待审核”。
5. 待审核期间，用户仍可以在自己的积木库中使用该积木。

规则：

- 只有积木所有者可以发布自己的积木。
- `private` 和 `rejected` 状态可以发布。
- `pending_review` 状态不能重复发布。
- `approved` 状态已经公开，按钮显示为“已公开”或进入分享页查看。

### 审核积木

1. 管理员进入待审核列表。
2. 管理员查看积木详情。
3. 管理员点击“通过”或“拒绝”。
4. 通过后状态变为 `approved`，公开展示。
5. 拒绝后状态变为 `rejected`，仅作者可见。

规则：

- 审核动作必须要求管理员角色。
- 拒绝原因本阶段先使用固定文案“未通过审核”，不做自定义原因输入。
- 审核通过不复制数据到新表，直接使用 `custom_blocks` 作为公开积木来源。

### 搜索公开积木

1. 用户打开“积木分享”。
2. 页面默认展示已审核通过的公开积木。
3. 用户可以输入关键词搜索名称、描述、分类、标签。
4. 用户可以按分类或标签筛选。
5. 用户可以按“最新”“热门”“适合新手”排序。
6. 用户点击积木卡片查看详情。

排序规则：

- 最新：按 `updated_at` 倒序。
- 热门：按 `favorite_count * 2 + import_count * 3 + view_count` 倒序。
- 适合新手：本阶段使用分类和标签的规则排序，优先展示分类为“动作”“风控”，或标签包含“新手”“基础”“止盈”“止损”的积木。

### 收藏公开积木

1. 登录用户在分享页点击“收藏”。
2. 系统创建收藏记录。
3. 用户再次点击可取消收藏。

规则：

- 游客可以浏览和搜索，但不能收藏。
- 同一用户对同一公开积木只能收藏一次。
- 收藏数展示在列表和详情中。

### 导入公开积木

1. 登录用户在公开积木详情或列表点击“导入到我的积木”。
2. 系统复制该公开积木的 `name`、`description`、`category`、`tags`、`template`。
3. 新积木归属于当前用户，状态为 `private`。
4. 如果当前用户已有同名积木，系统自动改名为 `原名（导入）`；如果仍冲突，继续追加数字，例如 `原名（导入 2）`。
5. 导入成功后，用户可以在搭建页积木库和个人空间中看到它。

规则：

- 用户可以导入自己的公开积木，但如果同名也按自动改名规则处理。
- 导入不会影响原公开积木。
- 导入数增加。

## 数据设计

继续使用 `custom_blocks` 作为积木主体表：

- `review_status`: `private`、`pending_review`、`approved`、`rejected`

新增表：

### `shared_block_stats`

用于存储公开积木热度数据。

- `id`
- `custom_block_id`
- `view_count`
- `favorite_count`
- `import_count`
- `created_at`
- `updated_at`

### `shared_block_favorites`

用于存储用户收藏。

- `id`
- `user_id`
- `custom_block_id`
- `created_at`

约束：

- `(user_id, custom_block_id)` 唯一。

### `shared_block_imports`

用于记录导入行为。

- `id`
- `user_id`
- `source_custom_block_id`
- `imported_custom_block_id`
- `created_at`

### `recommendation_events`

本阶段记录基础行为事件，后续推荐算法会使用。

- `id`
- `user_id`，游客为空
- `event_type`: `view`、`favorite`、`unfavorite`、`import`、`search`
- `custom_block_id`，搜索事件为空
- `keyword`，搜索事件填写
- `created_at`

## API 设计

### 个人积木发布

`POST /api/custom-blocks/{id}/publish`

权限：登录用户，只能发布自己的积木。

返回：更新后的 `CustomBlockResponse`。

错误：

- `404`: 积木不存在或不属于当前用户。
- `409`: 当前状态不能发布。

### 公开积木列表

`GET /api/shared-blocks?keyword=&category=&tag=&sort=&page=&pageSize=`

权限：游客可访问。

返回：

- `items`
- `total`
- `page`
- `pageSize`

每个 item 包含：

- 积木基础信息
- 作者用户名
- 节点数量
- 连接数量
- 收藏数
- 导入数
- 浏览数
- 当前用户是否已收藏

### 公开积木详情

`GET /api/shared-blocks/{id}`

权限：游客可访问。

行为：

- 增加浏览数。
- 记录 `view` 行为事件。

### 收藏与取消收藏

`POST /api/shared-blocks/{id}/favorite`

`DELETE /api/shared-blocks/{id}/favorite`

权限：登录用户。

### 导入公开积木

`POST /api/shared-blocks/{id}/import`

权限：登录用户。

返回：新创建的私有 `CustomBlockResponse`。

### 管理员审核

`GET /api/admin/custom-block-reviews?keyword=&page=&pageSize=`

`POST /api/admin/custom-block-reviews/{id}/approve`

`POST /api/admin/custom-block-reviews/{id}/reject`

权限：管理员。

## 前端设计

### 个人空间

在“我的积木”卡片动作区增加发布按钮：

- `private`: 显示“发布”
- `pending_review`: 显示“待审核”，按钮禁用
- `approved`: 显示“已公开”，可点击跳转分享详情页
- `rejected`: 显示“重新发布”

### 积木分享页

页面保留当前黑曜石质感，不做营销页。

布局：

- 顶部：搜索框、排序选择、分类筛选、标签筛选。
- 中部：公开积木列表卡片。
- 右侧或列表下方：详情面板，展示选中积木的信息和操作。
- 底部：分页。

卡片展示：

- 名称
- 作者
- 分类
- 标签
- 节点数和连接数
- 收藏数和导入数
- 按钮：查看、收藏、导入

### 管理员审核入口

本阶段先使用轻量入口：

- 如果当前用户有 `admin` 角色，积木分享页显示“审核”标签。
- 审核标签中展示待审核积木列表。
- 每条显示通过、拒绝按钮。

后续如果管理能力扩大，再拆出独立管理后台。

## 错误处理

- 游客点击收藏或导入：弹出登录弹窗。
- 发布失败：显示“发布失败，请稍后重试”。
- 重复发布：显示“该积木已提交审核或已公开”。
- 导入命名冲突：后端自动改名，前端显示最终名称。
- 管理员审核失败：显示“审核操作失败，请稍后重试”。

## 测试策略

后端测试：

- 用户只能发布自己的积木。
- `private` 和 `rejected` 可以发布。
- `pending_review` 和 `approved` 不能重复发布。
- 公开列表只返回 `approved` 积木。
- 关键词、分类、标签、排序、分页生效。
- 收藏幂等，不重复计数。
- 导入会复制模板并自动处理同名。
- 管理员可以通过或拒绝。
- 普通用户不能访问管理员审核接口。

前端测试：

- 个人空间显示发布按钮和状态。
- 点击发布后状态更新为待审核。
- 分享页加载公开积木列表。
- 搜索、筛选、排序会带正确查询参数。
- 收藏和取消收藏调用正确接口。
- 导入成功后显示新积木名称。
- 游客点击收藏/导入会触发登录弹窗。
- 管理员可看到审核标签并操作通过/拒绝。

浏览器验证：

- 个人空间发布按钮视觉和现有按钮一致。
- 分享页列表、筛选、详情面板在桌面宽度下不拥挤。
- 窄屏下筛选区和卡片不重叠。

## 实施顺序

1. 后端补发布、公开列表、详情、收藏、导入、审核接口。
2. 后端补数据表和服务层测试。
3. 个人空间增加发布按钮和状态。
4. 积木分享页实现列表、筛选、详情、收藏、导入。
5. 管理员审核标签页实现最小可用审核。
6. 全量测试、构建、浏览器验证。

## 验收标准

- 用户可以把自己的私有积木发布为待审核。
- 管理员可以通过该积木。
- 通过后的积木出现在积木分享页。
- 其他用户可以搜索、收藏并导入该积木。
- 导入后的积木出现在个人空间和搭建页积木库。
- 推荐能力只出现在积木分享页，不打扰搭建页。
