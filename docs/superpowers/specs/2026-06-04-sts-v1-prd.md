# STS V1 PRD 与实施说明

> STS 全称为 Simulated Trading System。本文档是 V1 产品需求文档，同时包含实施建议，强调每个模块“做什么”和“怎么一步一步实现”。

## 1. 项目概述

### 1.1 产品定位

STS 是一个面向量化交易学习者和普通投资者的模拟交易系统网站。用户即使不会写代码，也能通过类似积木拼接的方式创建盘中交易策略，并使用 A 股或美股的历史分钟级行情进行模拟回测，查看收益率、最大回撤、交易记录和买卖点走势图。

### 1.2 V1 核心目标

1. 打开网站后直达积木搭建界面。
2. 用户可以通过积木组合创建策略。
3. 用户可以创建模拟账户。
4. 用户可以选择 A 股或美股、股票、时间段、1 分钟或 5 分钟 K 线进行历史盘中回测。
5. 系统严格按真实市场规则模拟交易。
6. 用户可以保存策略、自定义参数化积木、管理个人内容。
7. 用户可以在论坛发帖、评论，并关联策略、回测结果或积木。
8. 用户可以在积木分享页搜索、筛选、收藏、导入公开积木。
9. 推荐算法只集成在积木分享页，不在搭建过程中弹出打扰用户。

### 1.3 技术要求

- 后端：Python 3.10+
- 数据库：MySQL 8.0
- 前端：Vue 3
- 架构：前后端分离
- 必备能力：增删改查、模糊查询、分页、JWT 令牌、文件上传下载、角色权限、推荐算法

### 1.4 V1 不做范围

1. 不接真实交易账户。
2. 不执行真实下单。
3. 不做实时自动交易。
4. 不做多股票组合策略主流程。
5. 不做卖空、杠杆、融资融券、期权、期货。
6. 不做复杂机器学习推荐模型。
7. 不在搭建界面主动推荐他人积木。

## 2. 用户角色与权限

### 2.1 角色

| 角色 | 说明 |
|---|---|
| 游客 | 未登录用户，可以浏览公开内容，可以在搭建页临时拼策略，但不能保存或运行完整回测 |
| 普通用户 | 注册登录用户，可以保存策略、回测、发帖、评论、收藏、导入积木 |
| 管理员 | 管理用户、内容、审核、标签、文件和推荐配置 |

### 2.2 权限规则

| 模块 | 游客 | 普通用户 | 管理员 |
|---|---|---|---|
| 浏览公开帖子 | 是 | 是 | 是 |
| 浏览公开积木 | 是 | 是 | 是 |
| 搭建临时策略 | 是，不保存 | 是 | 是 |
| 保存策略 | 否 | 是 | 是 |
| 创建模拟账户 | 否 | 是 | 是 |
| 运行回测 | 否，需登录 | 是 | 是 |
| 发帖 | 否 | 是，发布后默认待审核 | 是 |
| 评论 | 否 | 是，发布后默认待审核 | 是 |
| 收藏/导入积木 | 否 | 是 | 是 |
| 发布公开积木 | 否 | 是，发布后默认待审核；自动发布可由管理员配置 | 是 |
| 管理用户/内容 | 否 | 否 | 是 |

### 2.3 审核状态

普通用户点击“发布”后，前台仍显示发布动作，后台自动进入审核流程。用户不需要主动理解“提交审核”这个概念。

适用于帖子、评论、公开自定义积木。

| 状态 | 含义 |
|---|---|
| draft | 草稿，只有作者可见 |
| pending_review | 已发布，等待审核 |
| published | 审核通过，公开可见 |
| rejected | 审核不通过，作者可修改后再次发布 |
| hidden | 作者主动隐藏 |
| removed | 管理员下架 |

### 2.4 怎么实现

1. 先实现 `users`、`roles`、`user_roles`。
2. 登录成功后签发 JWT，JWT 中包含 `user_id` 和角色。
3. 后端每个需要登录的接口通过依赖函数解析 JWT。
4. 普通用户只能操作自己的私有资源。
5. 管理员接口统一使用 `/api/admin/*` 前缀。
6. 所有公开内容统一使用 `status` 字段控制可见性。

## 4. 积木搭建系统

### 4.1 设计原则

用户看到的是简单积木；系统内部必须使用类型化模型解释策略。

每个积木包含：

| 类型 | 说明 |
|---|---|
| 隐式输入 | 系统自动注入，如行情、账户、持仓、市场规则 |
| 用户参数 | 用户填写的值，如 5%、50%、10 根 K 线 |
| 输出 | 给下一个积木或回测引擎使用的结果 |

### 4.2 隐式输入上下文

| 输入 | 内容 |
|---|---|
| 行情数据 | 当前 K 线、历史 K 线、开高低收、成交量 |
| 账户状态 | 总资产、可用资金、冻结资金、收益率、最大回撤 |
| 持仓状态 | 持仓数量、持仓成本、可卖数量、当前仓位、持仓收益率 |
| 市场规则 | A 股/美股、T+1、最小交易单位、涨跌停、手续费 |
| 策略状态 | 冷却状态、上次买卖时间、最高浮盈、历史触发记录 |
| 当前时间 | 当前 K 线时间、是否开盘、是否收盘前 N 分钟 |

### 4.3 输出类型

| 输出类型 | 说明 |
|---|---|
| boolean | 条件是否成立 |
| number | 指标数值，如 RSI、收益率 |
| price | 价格值，如止盈价、止损价 |
| series | 指标序列，如 MA、MACD |
| trade_intent | 买入、卖出、清仓、调仓等交易意图 |
| control_effect | 冷却、暂停交易、记录状态等控制效果 |

### 4.4 必须积木

| 积木 | 规则 |
|---|---|
| 买入 | 买到目标仓位 X%，默认 30%，最大仓位默认 80% |
| 卖出 | 卖出持仓比例 Y%，默认 50% |
| 清仓 | 卖出当前股票全部可卖持仓 |
| 止盈 | 如果持仓收益率 >= X%，则卖出 Y% |
| 止损 | 如果持仓亏损率 >= X%，则卖出 Y% 或清仓 |
| 异常冷却 | 异常条件触发后，进入 N 根 K 线冷却 |

### 4.5 建议积木

| 积木 | 规则 |
|---|---|
| 移动止损 | 如果最高浮盈已达到 A%，且从最高浮盈回落 B%，则卖出 Y% |
| 调仓 | 将当前股票仓位调整到 X%，可覆盖加仓和减仓 |

### 4.6 条件与指标积木

V1 支持：

- 价格条件
- 涨跌幅条件
- 持仓收益率条件
- 仓位条件
- 均线条件
- 成交量条件
- 时间条件
- 冷却状态条件
- 且 / 或 / 非

V1 指标支持：

- MA
- RSI
- MACD
- 布林带
- VWAP

### 4.7 自定义积木

V1 采用参数化组合积木。

创建流程：

1. 用户拼出一段可运行逻辑。
2. 选中一组积木。
3. 点击“保存为自定义积木”。
4. 系统识别可参数化的值。
5. 用户选择暴露哪些参数。
6. 用户填写名称、描述、分类、标签。
7. 用户选择私有或公开。
8. 系统校验通过后保存。

V1 不允许用户写代码或脚本。

### 4.8 策略 JSON 建议结构

策略保存为 JSON，便于前端渲染和后端解释。

```json
{
  "version": "1.0",
  "blocks": [
    {
      "id": "condition-1",
      "type": "indicator_condition",
      "name": "RSI 条件",
      "params": {
        "indicator": "RSI",
        "period": 14,
        "operator": "<",
        "value": 30
      },
      "output": "boolean"
    },
    {
      "id": "action-1",
      "type": "buy_to_target_position",
      "params": {
        "targetPositionPercent": 50,
        "maxPositionPercent": 80
      },
      "output": "trade_intent"
    }
  ],
  "connections": [
    {
      "from": "condition-1",
      "to": "action-1"
    }
  ]
}
```

### 4.9 怎么实现

1. 先定义后端积木类型枚举和参数 schema。
2. 前端右侧积木库从后端读取内置积木定义。
3. 画布中每个积木实例保存为 JSON 节点。
4. 保存策略时，后端校验 JSON 结构、参数范围和类型连接。
5. 回测前，后端再次校验策略可运行性。
6. 自定义积木本质上保存一段子图 JSON 加参数定义。

## 5. 模拟账户与盘中回测

### 5.1 模拟账户

用户可以创建模拟账户，用于历史回测。

账户包含：

- 初始资金
- 当前总资产
- 可用资金
- 冻结资金
- 未交收资金
- 持仓
- 可卖持仓
- 交易记录

### 5.2 回测粒度

V1 支持：

- 5 分钟 K 线：默认周期
- 1 分钟 K 线：进阶短周期模式，受数据源可获取范围限制

系统允许后续扩展 15 分钟、30 分钟。

### 5.3 成交规则

| 场景 | 成交规则 |
|---|---|
| 普通买入 | 下一根 K 线开盘价成交 |
| 普通卖出 | 下一根 K 线开盘价成交 |
| 调仓 | 下一根 K 线开盘价成交 |
| 清仓 | 下一根 K 线开盘价成交 |
| 止盈 | 当前 K 线触达止盈价则按止盈价成交 |
| 止损 | 当前 K 线触达止损价则按止损价成交 |
| 移动止损 | 当前 K 线触达移动止损价则按触发价成交 |
| 同一根 K 线同时触发止盈和止损 | 默认优先按止损成交 |
| 冷却 | 从下一根 K 线开始生效 |

### 5.4 市场规则

V1 必须严格按真实市场规则模拟。

#### A 股

| 规则 | V1 处理 |
|---|---|
| T+1 | 当天买入股票当天不可卖出 |
| 做 T | 必须有隔夜底仓，只能卖出可卖持仓后买回 |
| 买入单位 | 默认 100 股整数倍 |
| 卖出单位 | 默认 100 股整数倍，零股按规则一次性卖出 |
| 涨跌停 | 主板普通股 10%，ST 5%，科创/创业板可配置 20% |
| 最小报价单位 | 0.01 元 |
| 融资融券/卖空 | V1 不做 |

#### 美股

| 规则 | V1 处理 |
|---|---|
| 日内交易 | 支持同日买卖 |
| 交收周期 | 普通证券按 T+1 交收 |
| 股数单位 | 默认 1 股起，不做碎股 |
| 涨跌停 | 不套用 A 股式每日涨跌停 |
| 交易时段 | 默认常规交易时段 9:30-16:00 ET |
| 卖空/杠杆 | V1 不做 |

### 5.5 回测结果

回测完成后输出：

- 总收益率
- 年化收益率，可选
- 最大回撤
- 胜率
- 交易次数
- 交易明细
- 资金曲线
- 持仓变化
- 买卖点走势图
- 未成交原因记录

### 5.6 怎么实现

1. 用户创建模拟账户。
2. 用户选择策略、账户、市场、股票、时间段、K 线周期。
3. 后端检查行情缓存是否覆盖该时间段。
4. 缺失数据由行情适配器按需获取并写入缓存。
5. 回测引擎加载策略 JSON。
6. 回测引擎逐根 K 线推进。
7. 每根 K 线注入行情、账户、持仓、市场规则、策略状态。
8. 策略解释器生成 `trade_intent` 或 `control_effect`。
9. 市场规则校验器判断是否允许成交。
10. 成交模拟器计算成交价、手续费、滑点、股数取整。
11. 账户引擎更新资金、持仓、可卖数量。
12. 指标计算器记录收益率、回撤、资金曲线。
13. 回测结果写入数据库并返回图表数据。

## 6. 行情数据方案

### 6.1 原则

考虑电脑内存有限，V1 不提前下载全市场数据。

采用：

1. 用户按需选择股票和时间段。
2. 后端只获取该股票、该时间段、该周期的数据。
3. 获取后缓存到 MySQL。
4. 下次回测优先读取缓存。
5. 缺哪段补哪段。

### 6.2 数据源

V1 建议：

- A 股：AKShare
- 美股：AKShare 或 yfinance

数据源能力不稳定时，系统应提示用户缩短时间范围、切换周期或稍后重试。

### 6.3 K 线缓存字段

建议统一为：

- market
- symbol
- name
- interval
- timestamp
- open
- high
- low
- close
- volume
- amount
- currency
- source
- created_at

### 6.4 怎么实现

1. 定义 `MarketDataProvider` 接口。
2. 实现 `AkshareProvider`。
3. 可选实现 `YFinanceProvider` 作为美股备用。
4. 后端统一输出标准 OHLCV 数据。
5. 缓存表对 `market + symbol + interval + timestamp` 建唯一索引。
6. 回测前调用 `ensure_kline_data()` 检查并补齐数据。

## 7. 论坛与积木分享

### 7.1 论坛

论坛发帖是统一流程：

1. 写标题。
2. 选择帖子类型。
3. 选择标签。
4. 写正文。
5. 可选关联策略、回测记录、积木。
6. 可选上传图片或附件。
7. 点击发布。
8. 后台进入审核。
9. 审核通过后公开。

帖子类型：

- 普通讨论
- 策略分享
- 回测分享
- 积木经验

帖子可关联对象：

- 策略
- 回测记录
- 自定义积木
- 公开积木

### 7.2 积木分享

积木分享页是公开积木的搜索、筛选、推荐和导入中心。

能力：

- 公开积木列表
- 关键词搜索
- 标签筛选
- 分类筛选
- 输入输出类型筛选
- 热门/最新/推荐排序
- 积木详情
- 收藏
- 导入到我的积木库

### 7.3 怎么实现

1. 论坛和积木分享分表保存，避免讨论内容和可复用资产混在一起。
2. 帖子通过 `related_type`、`related_id` 关联策略、回测或积木。
3. 公开积木详情页展示参数定义、输入输出类型、作者、使用次数、收藏数。
4. 用户点击导入后，复制公开积木定义到自己的自定义积木库。
5. 用户在论坛帖子中点击积木卡片，跳转到积木分享详情页。

## 8. 推荐算法

### 8.1 产品边界

推荐功能只集中在积木分享页。

不在搭建积木页弹出推荐，不打断用户搭建策略。

论坛只支持用户主动附加积木链接；帖子详情中点击积木卡片后跳转到积木分享详情页。

### 8.2 推荐类型

| 类型 | 说明 |
|---|---|
| 热门积木 | 按收藏数、导入数、使用次数、浏览量排序 |
| 相似积木 | 根据标签、分类、输入输出类型相似 |
| 个性化推荐 | 根据用户浏览、收藏、导入历史 |
| 新手精选 | 管理员精选基础积木 |
| 最新发布 | 按发布时间展示 |

### 8.3 推荐公式

登录用户：

```text
推荐分 = 0.35 * 标签相似度
       + 0.25 * 热度分
       + 0.20 * 用户兴趣分
       + 0.10 * 质量分
       + 0.10 * 新鲜度分
```

游客：

```text
推荐分 = 0.50 * 热度分
       + 0.30 * 标签相似度
       + 0.20 * 新鲜度分
```

### 8.4 怎么实现

1. 记录用户行为事件：浏览、收藏、导入、搜索、点击标签。
2. 为公开积木维护标签、分类、输入输出类型、热度统计。
3. 查询积木分享列表时，根据排序模式计算推荐分。
4. 个性化推荐只对登录用户启用。
5. 管理员可配置新手精选。
6. V1 推荐只推荐积木，不推荐股票和买卖建议。

## 9. 数据模型与 API 模块

### 9.3 API 风格

采用 RESTful API。

统一约定：

- 列表接口支持分页。
- 列表接口支持关键词模糊查询。
- 列表接口支持排序。
- 需要登录的接口必须携带 JWT。
- 管理员接口使用 `/api/admin/*`。

示例：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/strategies?page=&keyword=`
- `POST /api/strategies`
- `GET /api/blocks`
- `POST /api/custom-blocks`
- `POST /api/backtests`
- `GET /api/backtests/{id}`
- `GET /api/shared-blocks?keyword=&tag=&sort=`
- `POST /api/forum/posts`
- `POST /api/files/upload`
- `POST /api/admin/audits/{id}/approve`

## 10. 技术架构

### 10.1 前端

建议：

- Vue 3
- Vue Router
- Pinia
- Axios
- ECharts
- 积木画布可后续选择 Blockly 或自定义 Vue 画布组件

### 10.2 后端

建议：

- Python 3.10+
- FastAPI
- SQLAlchemy
- Alembic
- Pydantic
- PyJWT 或 python-jose
- AKShare
- yfinance 作为美股备用

### 10.3 数据库

MySQL 8.0。

重点索引：

- 用户名、邮箱唯一索引
- 策略 owner_id 索引
- 公开内容 status 索引
- K 线缓存 `market + symbol + interval + timestamp` 唯一索引
- 论坛标题、正文、标签支持模糊查询

### 10.4 文件存储

V1 先使用本地文件存储：

- 图片
- 附件
- 回测报告

MySQL 保存文件元数据：

- 文件名
- 路径
- 类型
- 大小
- 上传者
- 关联业务对象
- 权限

### 10.5 后台任务

V1 可先使用轻量异步任务。

如果回测耗时明显，再升级 Redis + Celery。

PRD 中要求支持：

- 创建回测任务
- 查询任务状态
- 查询回测结果
- 失败时记录错误原因

## 11. 非功能需求

### 11.1 安全

- 密码必须加密存储。
- JWT 过期时间可配置。
- 用户只能访问自己的私有数据。
- 文件下载必须校验权限。
- 管理员操作写入审计日志。

### 11.2 性能

- 行情按需获取，不全量下载。
- K 线缓存必须复用。
- 列表接口必须分页。
- 回测任务应异步执行或至少提供任务状态。

### 11.3 可解释性

回测结果必须展示：

- 每次交易触发的积木
- 成交价格
- 未成交原因
- 市场规则限制原因

### 11.4 用户体验

- 搭建页不弹出推荐。
- 参数输入必须有默认值和范围提示。
- 回测失败时提示具体原因。
- 1 分钟数据不可用时提示缩短时间范围或切换 5 分钟。

## 12. V1 分阶段实施路线

### 阶段 0：项目骨架

目标：搭好前后端分离基础。

步骤：

1. 创建后端 FastAPI 项目。
2. 创建前端 Vue 3 项目。
3. 配置 MySQL 连接。
4. 配置环境变量。
5. 建立统一 API 返回格式。
6. 建立基础目录结构。

验收：

- 前端能访问后端健康检查接口。
- 后端能连接 MySQL。

### 阶段 1：用户、JWT、布局

目标：完成登录注册和基础页面框架。

步骤：

1. 实现用户注册、登录、密码加密。
2. 实现 JWT 签发和鉴权。
3. 实现游客、普通用户、管理员角色。
4. 前端实现左侧导航、顶部栏、登录注册页面。
5. 登录后显示用户信息。

验收：

- 用户可以注册登录。
- 未登录用户不能保存策略。
- 管理员可访问管理入口。

### 阶段 2：积木定义与策略保存

目标：实现策略 JSON 的创建、保存、加载。

步骤：

1. 建立内置积木定义表。
2. 写入动作、条件、指标、风控积木定义。
3. 前端展示右侧积木库。
4. 前端支持在画布创建、编辑、删除积木。
5. 后端校验策略 JSON。
6. 实现策略增删改查、分页、模糊查询。

验收：

- 用户可以拼出简单策略并保存。
- 用户可以在个人空间加载策略。

### 阶段 3：自定义积木

目标：实现参数化组合积木。

步骤：

1. 用户选中一组积木。
2. 前端识别可参数化字段。
3. 用户配置参数名、默认值、范围。
4. 后端保存自定义积木 JSON。
5. 用户可在自己的积木库中复用。

验收：

- 用户可以创建“如果收益率 >= X%，卖出 Y%”这类自定义积木。

### 阶段 4：模拟账户与市场规则

目标：完成账户、持仓、市场规则校验。

步骤：

1. 创建模拟账户表。
2. 创建持仓表和交易记录表。
3. 实现 A 股市场规则配置。
4. 实现美股市场规则配置。
5. 实现可卖持仓计算。
6. 实现成交前规则校验。

验收：

- A 股当天买入不能当天卖出。
- 美股支持同日买卖。
- 交易单位和资金限制生效。

### 阶段 5：行情数据与缓存

目标：按需获取 A 股/美股分钟数据。

步骤：

1. 定义行情适配器接口。
2. 实现 AKShare A 股数据获取。
3. 实现 AKShare 或 yfinance 美股数据获取。
4. 实现 K 线缓存表。
5. 实现缺失时间段补齐。

验收：

- 用户选择股票、时间段、周期后，系统能获取并缓存 K 线数据。

### 阶段 6：回测引擎

目标：完成从策略到回测结果的闭环。

步骤：

1. 实现策略解释器。
2. 实现指标计算器。
3. 实现交易意图生成。
4. 实现成交模拟。
5. 实现收益率和最大回撤计算。
6. 保存回测任务、成交明细、资金曲线。
7. 前端使用 ECharts 展示走势、买卖点和资金曲线。

验收：

- 用户能运行 1 分钟或 5 分钟历史盘中回测。
- 结果展示收益率、最大回撤、交易记录、买卖点图。

### 阶段 7：个人空间

目标：完成个人内容管理。

步骤：

1. 我的策略列表。
2. 我的自定义积木列表。
3. 我的模拟账户列表。
4. 我的回测记录。
5. 我的收藏。
6. 文件上传下载。

验收：

- 所有列表支持分页、模糊查询。
- 用户只能查看自己的私有内容。

### 阶段 8：论坛与审核

目标：完成论坛和内容审核。

步骤：

1. 实现帖子 CRUD。
2. 实现评论 CRUD。
3. 实现帖子关联策略、回测、积木。
4. 实现点赞和收藏。
5. 实现帖子、评论审核状态。
6. 管理员可以审核、下架内容。

- 管理员审核通过后公开展示。

### 阶段 9：积木分享与推荐

目标：完成公开积木搜索、导入和推荐。

## 13. V1 验收标准

V1 完成时，必须能演示以下闭环：

1. 用户注册并登录。
2. 用户进入搭建积木页。
3. 用户使用条件、指标、动作、风控积木搭建策略。
4. 用户保存策略。
5. 用户创建模拟账户。
6. 用户选择 A 股或美股、股票、时间段、1 分钟或 5 分钟 K 线。
7. 系统按需获取行情数据。
8. 用户运行回测。
9. 系统展示收益率、最大回撤、交易记录和买卖点走势图。
10. 用户保存自定义积木。
11. 用户发布帖子并关联回测结果。
12. 管理员审核帖子。
13. 用户在积木分享页搜索、收藏、导入公开积木。
